# Teammate 工具 0.4.4 行为契约

2026-07-15 对 teammate 工具族（teammate/-send/-list/-watch）做交叉审计（文档 vs schema vs 实现），修复 F1–F11，全记录见 `docs/teammate-tool-fix-plan.md` 与 `packages/pi-maestro-teammate/CHANGELOG.md`。

## 决策裁定

| # | 问题 | 裁定 |
|---|------|------|
| ① | reply_to 死锁检测（死代码 + 假 fallback） | 整段删除。schema 维持 `caller\|main`；若未来需要 agent 名定向回复路由需重新设计 |
| ② | 多任务 `context: "fork"` 被静默丢弃 | 透传生效，TaskSpec 支持 per-task 覆盖。fork N 任务 = N 份父会话拷贝 |
| ③ | `protocol_version` 暴露给 LLM | 从对外 schema 移除，运行时保留兼容（pi 校验允许未知属性） |
| ④ | 依赖只靠 {name} 插值 | 引入 `dependsOn`，依赖边 = {name} 推导 ∪ dependsOn，未知名严格报错 |
| ⑤ | abort 塞在 send 里 | 不拆分；`message` 对 abort 可选，mode 默认 follow_up 写入 schema |

## Breaking 行为变化（0.4.3 → 0.4.4）

1. `tasks + context:"fork"` 从静默 fresh 变为真 fork（token 成本上升）。
2. 空任务（task 与 prompt 均缺）从空跑变为派发前报错。
3. `chain + tasks` 同给时优先级反转：tasks 生效（原 chain 生效）；chain 任何使用都有弃用警告，计划下个 minor 移除。

## 引用校验策略（F2）

- 未知 `{ref}` 与现有任务名编辑距离 ≤（len≤3 ? 1 : 2）→ 视为拼写错误，**派发前报错**；
- 其余未知 `{ref}` → `[warn]` 警告后按字面文本传递；
- 全部任务无 name 时跳过分析（引用意图不可能存在）。

## teammate vs maestro 选择判据

两者共用 runTeammate 引擎。派发 pi agent 角色（agents/*.md，DAG/结构化输出/fork）→ teammate；调用外部 CLI endpoint（gemini/codex 等并行搜索、委托、MoA）→ maestro。
